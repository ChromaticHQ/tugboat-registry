Can Tugboat find your document root?

Tugboat looks for a folder named docroot in your repository to serve content from. This is a common pattern, but many projects have their own preference. If your document root is in a different location you have a couple of different options.

Option 1. Add a docroot/ folder to your repository

This is the most straightforward fix for Tugboat, but can be problematic if your repository has already been deployed elsewhere. If you choose this option, create a docroot folder and add your repository's web content. Also, If your site is deployed elsewhere, make sure each deployment still works as expected.

Remember to rebuild the ">preview after you make any changes to your build script. This will happen automatically for previews built for pull requests (or merge requests), but needs to be done manually for other previews.

Option 2. Create a build script

Build scripts can be used to perform any number of customizations to your preview. But, if Tugboat just needs to look in a different folder within your repository, a simple script like the following may be all you need. Add the following lines to a file named Makefile in the root of your repository. Make sure the indent is a TAB, and not multiple spaces. 

tugboat-init:
	ln -sf $TUGBOAT_ROOT/public_html /var/www/html

The above example instructs Tugboat to use the public_html folder inside your repository rather than docroot. Be sure to change the folder name if your directory structure is different.

Remember to rebuild the ">preview after you make any changes to your build script. This will happen automatically for previews built for pull requests (or merge requests), but needs to be done manually for other previews.
