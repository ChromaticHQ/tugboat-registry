Can Tugboat find your document root?

Tugboat looks for a folder named docroot in your repository to serve content from. This is a common pattern, but many projects have their own preference. If your document root is in a different location you may want to add a docroot/ folder to your repository or use a build script to customize Tugboat.

Add a docroot/ folder to your repository

This is the most straightforward fix for Tugboat, but can be problematic if your repository has already been deployed elsewhere. If you choose this option, create a docroot folder and add your repository's web content. Also, If your site is deployed elsewhere, make sure each deployment still works as expected.

Remember to rebuild the ">preview after you make any changes! This will happen automatically for previews built for pull requests (or merge requests), but needs to be done manually for other previews.
