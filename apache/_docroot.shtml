Can Tugboat find your document root?

Tugboat looks for a folder named docroot in your repository to serve content from. This is a common pattern, but many projects have their own preference. If your document root is in a different location you have a couple of different options.

Option 1. Add a docroot/ folder to your repository

This is the most straightforward fix for Tugboat, but can be problematic if your repository has already been deployed elsewhere. If you choose this option, create a docroot folder and move your existing content you want Apache to serve into that folder. Be sure to follow up on other deployments to make sure things still work.

Remember to rebuild the ">preview after you make any changes to your build script. This will happen automatically for previews built for pull requests (or merge requests), but needs to be done manually for other previews.

Option 2. Create a build script.

Build scripts can be used to perform any number of customizations to your preview. But, if all you need is to point Apache to a different location in your repository, a simple script like the following may be all you need. Add this to a file named Makefile in the root of your repository. Make sure the indent is a TAB, and not multiple spaces. 

tugboat-init:
	ln -sf $TUGBOAT_ROOT/public_html /var/www/html

If the content you want to serve Apache lives in a folder other than public_html, be sure to change the folder name.

Remember to rebuild the ">preview after you make any changes to your build script. This will happen automatically for previews built for pull requests (or merge requests), but needs to be done manually for other previews.
