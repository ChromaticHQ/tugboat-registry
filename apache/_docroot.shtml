Can Tugboat find your document root?

By default, Tugboat attempts to serve content from a /docroot folder in your repository. This is a common strategy, but many projects have their own preference.

If the web content of this repository is in a different location, moving it to a /docroot folder is the most straightforward fix for Tugboat. However, this can be problematic if the repository is already deployed elsewhere. Double-check that other deployments still work as expected if this route is chosen.

The alternative to moving content around in the repository is to create a build script. Read below for details.
