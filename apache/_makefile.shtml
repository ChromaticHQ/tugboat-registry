Customizing Tugboat with a build script

Tugboat uses GNU Make to provide the opportunity to run your own scripts at different points of a preview's life cycle. It is very likely that your repository will need a build script in order to work correctly with Tugboat. With build scripts you can change the location of your docroot folder, import databases and image assets, run test scripts and programmically configure your services.

Example Build Script: Linking to a different Document Root

If Tugboat just needs to look in a different folder within your repository to serve web content from, a simple script like the following may be all you need. Add the following lines to a file named Makefile in the root of your repository. Make sure the indent is a TAB, and not multiple spaces. 

      tugboat-init:
      	ln -sf $TUGBOAT_ROOT/public_html /var/www/html

The above example instructs Tugboat to use the public_html folder inside your repository rather than docroot. If you copy this example, be sure to change the folder name if your directory structure is different.

Remember to rebuild the ">preview after you make any changes to your build script. This will happen automatically for previews built for pull requests (or merge requests), but needs to be done manually for other previews.

Check out our documentation for more information on creating a Tugboat build script.
