Is your project in a /docroot folder?

Your web server, Apache, is looking for your project in /docroot. If you don’t have a docroot folder in your repository, either change your project folder structure or create a Makefile build script to specify the default folder for the web server (e.g. public_html):

tugboat-init:
  ln -s public_html docroot

Remember to rebuild preview after you create the makefile. Any time you create a pull request preview, it’s updated automatically; branches and tags need to be updated manually to see changes.
